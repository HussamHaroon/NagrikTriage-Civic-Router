# PromptWars × Global Prompt Challenge — Rules & Context

**Host:** DEVENGERS · Hack2Skill · Google for Developers
**Event:** Solo, online, AI-powered build challenge
**Date:** 7 July 2026
**Build window:** 10:30 AM IST → 2:30 PM IST (4 hours, hard cap)
**Pitch round (Top 10):** 4:00 PM – 5:00 PM IST
**Result announcement:** After 5:00 PM IST

> This document is a **pre-event brief**. Nothing will be built until the problem statement is released on 7 July at 10:30 AM IST.

---

## 1. Eligibility & Format

- **Mode:** Online only (Google Meet).
- **Team size:** Solo (no pair / team submissions).
- **Entry fee:** Free.
- **Required registration:** Hack2Skill platform + DEVENGERS WhatsApp community.
- **Required hardware:** Laptop/desktop + stable internet connection.
- **Registered:** Prior to Mon 6 Jul 2026 (registration deadline).

## 2. The 4-Hour Rule (CRITICAL)

The competition is a **hard 4-hour build window**:

| Phase | Time | What happens |
|---|---|---|
| Opening + Briefing | 10:00 – 10:30 AM | Rules + problem statement released |
| **Build window starts** | **10:30 AM** | Problem statement live |
| **Build window ends** | **2:30 PM SHARP** | **All submissions closed** |
| Top 10 Pitch | 4:00 – 5:00 PM | Finalists demo to jury |
| Results | After 5:00 PM | Winners announced |

⚠️ **Late submissions are not accepted, under any circumstances.** Plan the last 30 minutes for *submission only*, not coding.

## 3. Mandatory Submission Artifacts (all 4 required)

Every submission must include **all four** of these, working and accessible:

1. **Public GitHub Repository Link**
   - Must be `public` (not private).
   - Must have a meaningful commit history.
   - Must contain a `README.md`.

2. **Deployed Project / Demo Link**
   - Must be live and reachable.
   - Recommended platforms: Vercel, Netlify, Render, Railway, Replit, or similar.
   - The deployed app must reflect the latest code in the repo.

3. **Project Title + Description**
   - 2–4 sentence summary of what it does and the problem it solves.

4. **Prompt Workflow / Strategy**
   - The exact prompts and AI tools you used.
   - How the AI slots into the product.
   - Format: can be a section in the README or a separate `PROMPTS.md`.

## 4. Allowed Tools

- Google Gemini (and any Google AI tool)
- Any LLM / generative AI tool
- Prompt engineering platforms
- No-code / low-code platforms (Lovable, Bolt, v0, Bubble, FlutterFlow, Glide, …)
- APIs (free tier is fine)
- Automation / workflow tools

❌ **Not allowed:**

- Submitting work started before 10:30 AM on 7 July 2026.
- Using pre-built projects and pretending they're new.
- Submitting someone else's code (with or without credit does not count).
- Submitting an empty repo, "lorem ipsum," or anything non-functional.
- Links that 404, are private, or require auth.
- Submitting after 2:30 PM IST (no exceptions).

## 5. Judging Criteria (used by the AI jury and human jury)

Final scoring will be against:

1. **Innovation** — originality and creativity
2. **Code quality** — structure, readability, hygiene
3. **Problem alignment** — fit between problem, solution, and execution
4. **AI usage** — depth and authenticity of AI integration
5. **Usability** — real-world readiness, UX, accessibility
6. **Performance** — speed, reliability, robustness
7. **Overall impact** — scale, usefulness, measurable effect

For Top 10 finalists, the human jury also evaluates pitch delivery, demo fluency, and Q&A.

## 7. Disqualification Conditions

The organizer may disqualify any submission that:

- Misses the 2:30 PM deadline.
- Has a broken/dead/private repo or deployment link.
- Is found to be a prior project resubmitted as new.
- Contains plagiarism or unattributed third-party code that is itself the *substance* of the submission.
- Violates the spirit of "build during 10:30–2:30" in any way.
